# Claude-Opus — Round 4: The Translation Layer Is the Whole Game

Qwen's R3 test is good but it **hides the hardest problem behind a convenience**: every triple used the *same* relation `rel` with `shift=1`. That makes the unbind trivial because there's no domain-crossing. The real unsolved problem is the **count↔XOR boundary**, and I'm going to nail it precisely, including the one place where it's provably lossy.

## First: state the two algebras exactly, so the boundary is unambiguous

```rust
/// XOR domain: an atom is its EXACT bit pattern. Invertible.
/// a XOR b XOR b = a.  Single-hop, exact composites.
pub struct BinaryAtom {
    bits: [u64; D / 64],     // D=16384 -> 256 u64s, 2KB
}

/// COUNT domain: a bundle is a histogram. Lossy, statistical.
/// Recovery is by cleanup (nearest-codebook), NOT by inversion.
pub struct CountCentroid {
    counts: [u8; D],
    n_items: u32,
    l1_norm: u32,
}
```

**The fundamental asymmetry nobody has stated cleanly:**

- **XOR unbind is exact and algebraic.** `unbind(c, role) = c XOR role`. You get bits back.
- **Count unbind is NOT inversion.** Permutation unbind `ρ⁻¹` recovers the *histogram*, but the histogram is a superposition. You must run **cleanup** (codebook nearest-neighbor) to get a discrete atom back.

So the boundary has a hard direction-dependence:

```
COUNT -> XOR :  REQUIRES cleanup (a "measurement" / collapse).  Lossy. SNR-gated.
XOR -> COUNT :  free (just fold the bits in as a count increment). Lossless.
```

This is the crux. **The translation layer is not a function — it's a measurement.** You cannot pass a count-bundle into an XOR unbind directly; you must first *collapse* it to a discrete `BinaryAtom` via cleanup. Let me show why, then show the pipeline.

## Why you cannot XOR-unbind a count vector (the lossy direction, derived)

Suppose the frontier centroid holds `counts`, and you want to unbind relation `R` to get next-hops. The naive idea: convert counts→bits (threshold), then XOR with `R`. **This is wrong** and here's the exact failure:

A count position `counts[j] = m_j` encodes *how many bundled triples lit bit j*. Thresholding to a bit `b_j = [m_j > τ]` throws away `m_j`. But the multiplicity `m_j` IS the signal — it's what separates a real next-hop (lit by `m` agreeing triples) from background (lit by ~2Np permuted-noise triples). XOR operates on `b_j ∈ {0,1}`, so **the moment you threshold, you're back in the N≤107 binary-capacity regime.** You've thrown away the very thing that bought you C_max>1000.

**Conclusion: the unbind in the count domain must be done by permutation (preserving counts), and the collapse to XOR happens only at the *final* cleanup step, per-candidate.** Here's the full pipeline.

Goal: `start --R1--> {h1} --R2--> {h2} --R3--> {h3}`, where each hop fans out and the frontier grows.

### Step 1 — HOP: unbind R in the COUNT domain, then collapse via cleanup

This is the load-bearing function. **Three phases: permute-unbind (count, lossless), score-against-codebook (the measurement), collapse-to-atoms (the COUNT→XOR boundary crossing).**

```rust
impl Frontier {
    fn hop(&self, rel: &Relation, codebook: &AtomMemory) -> Frontier {
        // ---- PHASE A: count-domain unbind (permutation, NO thresholding) ----
        // For each centroid, ρ_R⁻¹ applied to counts. Counts survive intact.
        // This is the ONLY correct unbind in the count domain.
        let mut all_candidates: Vec<Candidate> = Vec::new();

        for c in &self.bundle.centroids {
            let unbound: [u8; D] = rel.inv.apply_counts(&c.counts);  // [u8;D] -> [u8;D]

            // ---- PHASE B: the MEASUREMENT (count -> candidate atoms) ----
            // Background per position (the analytic subtraction from R3):
            let bg = (c.l1_norm as f32) * 2.0 / (D as f32);

            // Drive cleanup from high-count positions via inverted index.
            let thresh = (bg + 3.0 * bg.sqrt()).ceil() as u8;
            let mut hits: HashMap<u32, i32> = HashMap::new();
            for pos in 0..D {
                let cnt = unbound[pos];
                if cnt < thresh { continue; }
                for &aid in &codebook.posting[pos] {
                    *hits.entry(aid).or_insert(0) += cnt as i32;
                }
            }
            // Background-correct: subtract k*bg (expected overlap from noise).
            for (aid, raw) in hits {
                let score = raw as f32 - (K as f32) * bg;
                if score > 0.0 { all_candidates.push(Candidate { aid, score }); }
            }
        }

        // ---- PHASE C: COLLAPSE (count -> XOR). THE BOUNDARY CROSSING. ----
        // Aggregate scores across centroids, take survivors above SNR floor.
        // Each survivor is now a DISCRETE BinaryAtom fetched from codebook.
        // THIS is where count-domain becomes XOR-domain. The collapse is
        // a measurement: we replace a superposed histogram with concrete bits.
        let survivors = aggregate_and_threshold(all_candidates, /*snr*/ 5.0);

        let cleaned: Vec<(BinaryAtom, f32)> = survivors.iter()
            .map(|s| (codebook.atoms[s.aid as usize].to_binary(), s.score))
            .collect();

        // ---- PHASE D: REBUILD the count bundle for the NEXT hop ----
        // Fold the cleaned discrete atoms back into count centroids
        // (XOR -> count, the FREE direction). Re-cluster into M centroids.
        let next_bundle = FrontierBundle::rebuild_from_atoms(&cleaned, M);

        Frontier { bundle: next_bundle, atoms: cleaned }
    }
}
```

### The boundary, stated as a theorem

**The COUNT→XOR crossing happens exactly once per hop, at Phase C, and it is a measurement (collapse), not an inverse.** Properties:

1. **It is SNR-gated.** Only candidates with aggregate score > 5σ survive. Below that, the atom is *lost* — there is no recovering it, because the histogram genuinely doesn't distinguish it from noise. This is the irreducible lossiness.

2. **It is idempotent on clean atoms.** A survivor `a` re-folded into counts (Phase D) and unbound again returns `a`. So the pipeline is stable across hops *for atoms that survive the gate.*

3. **The XOR→COUNT direction (Phase D) is lossless and free** — `counts[j] += 1` for each set bit. No SNR cost. This asymmetry is why we rebuild rather than carry a running count vector through XOR ops.

**The Boundary Theorem (stated precisely):**

- **collapse** (COUNT→XOR): `CountCentroid -> Vec<BinaryAtom>`. Operation: unbind via permutation, score against codebook, threshold at SNR ≥ 5σ, fetch discrete atoms. **Lossy, O(M·D + M·H·L), SNR-gated.**

## 2. The Translation Layer: Exact Operations

```rust
impl Frontier {
    /// XOR→COUNT: Fold discrete atoms into count centroids.
    /// Used when transitioning from XOR path back to count path.
    pub fn fold_into_bundle(&mut self) {
        let mut centroids = [CountCentroid::zero(); 8];
        
        // Simple round-robin assignment (online k-means is overkill for rebuild)
        for (i, (atom, _score)) in self.atoms.iter().enumerate() {
            let c_idx = (i % 8) as u8;
            let c = &mut centroids[c_idx as usize];
            for j in atom.iter_set_bits() {
                c.counts[j] = c.counts[j].saturating_add(1);
            }
            c.n_items += 1;
            c.l1_norm += 64; // K=64 bits per atom
        }
        
        self.bundle = FrontierBundle {
            centroids,
            member_ids: (0..self.atoms.len() as u64).collect(),
            assign: (0..self.atoms.len()).map(|i| (i % 8) as u8).collect(),
        };
    }

    /// COUNT→XOR: Collapse count bundle to discrete atoms via cleanup.
    /// This is the MEASUREMENT. Lossy. Returns survivors above SNR floor.
    pub fn collapse_from_bundle(
        &self,
        rel: &Relation,
        codebook: &AtomMemory,
    ) -> Vec<(BinaryAtom, f32)> {
        let mut candidates: HashMap<u32, f32> = HashMap::new();

        for c in &self.bundle.centroids {
            // Unbind via inverse permutation (count-domain, lossless)
            let unbound = rel.inv.apply_counts(&c.counts);
            
            // Background subtraction
            let bg = (c.l1_norm as f32) * 2.0 / 16384.0;
            let thresh = (bg + 3.0 * bg.sqrt()).ceil() as u8;

            // Inverted-index scan
            for pos in 0..16384 {
                if unbound[pos] < thresh { continue; }
                for &aid in &codebook.posting[pos] {
                    *candidates.entry(aid).or_insert(0.0) += unbound[pos] as f32;
                }
            }
        }

        // Aggregate, background-correct, threshold at SNR ≥ 5
        let bg_global = self.bundle.centroids.iter()
            .map(|c| (c.l1_norm as f32) * 2.0 / 16384.0)
            .sum::<f32>() / 8.0;

        candidates.into_iter()
            .map(|(aid, raw)| {
                let score = raw - 64.0 * bg_global;
                (aid, score)
            })
            .filter(|(_, score)| *score > 5.0 * bg_global.sqrt()) // SNR ≥ 5σ
            .map(|(aid, score)| (codebook.atoms[aid as usize].to_binary(), score))
            .collect()
    }
}
```

```rust
    /// Uses the statistical `bundle` to generate a new set of discrete `atoms`.
    /// This is the SNR-gated bottleneck.
    pub fn collapse(&mut self, rel: &Relation, codebook: &AtomMemory) {
        let mut candidates: HashMap<u32, (u32, f32)> = HashMap::new(); // aid -> (multiplicity, score)
        let bg_global = self.bundle.centroids.iter()
            .map(|c| (c.l1_norm as f32) * 2.0 / (D as f32))
            .sum::<f32>() / M as f32;
        
        // Corrected noise sigma: The score is a sum over K positions, so the variance of the noise
        // contribution to the score is K * Var(noise_per_pos) = K * bg.
        // The standard deviation is thus sqrt(K * bg). This is a critical correction.
        let noise_sigma = (K as f32 * bg_global).sqrt();
        let snr_gate = 5.0 * noise_sigma; // 5σ gate on the *total score*

        for c in &self.bundle.centroids {
            let unbound = rel.inv.apply_counts(&c.counts);
            let bg_centroid = (c.l1_norm as f32) * 2.0 / (D as f32);
            let thresh = (bg_centroid + 3.0 * bg_centroid.sqrt()).ceil() as u8;

            for pos in 0..D {
                if unbound[pos] < thresh { continue; }
                for &aid in &codebook.posting[pos] {
                    let entry = candidates.entry(aid).or_insert((0, 0.0));
                    entry.0 += 1; // Increment hit count (proxy for multiplicity)
                    entry.1 += unbound[pos] as f32; // Accumulate raw count score
                }
            }
        }

        let mut survivors = Vec::new();
        for (aid, (multiplicity, raw_score)) in candidates {
            let score = raw_score - (K as f32 * bg_global); // Background subtraction
            if score > snr_gate {
                survivors.push((
                    codebook.atoms[aid as usize].to_binary(),
                    multiplicity,
                    score,
                ));
            }
        }
        self.atoms = survivors;
    }

    /// TRANSITION 2: `fold` (XOR -> COUNT). The lossless projection.
    /// Rebuilds the statistical `bundle` from the discrete `atoms`.
    pub fn fold(&mut self) {
        let mut new_bundle = FrontierBundle { centroids: std::array::from_fn(|_| CountCentroid::zero()) };
        for (i, (atom, _, _)) in self.atoms.iter().enumerate() {
            let centroid_idx = i % M;
            let c = &mut new_bundle.centroids[centroid_idx];
            for j in atom.iter_set_bits() {
                c.counts[j] = c.counts[j].saturating_add(1);
            }
            c.n_items += 1;
            c.l1_norm += K as u32;
        }
        self.bundle = new_bundle;
    }
}
```

```rust
impl FrontierBundle {
    fn hop(&self, rel: &Relation, atoms: &AtomMemory) -> Vec<(BinaryAtom, f32)> {
        let mut candidates: HashMap<u32, f32> = HashMap::new();
        
        for c in &self.centroids {
            // COUNT-domain unbind: permutation (no thresholding)
            let unbound = rel.perm.inv().apply_counts(&c.counts);
            
            // Cleanup against codebook (inverted-index scan)
            let bg = (c.l1_norm as f32) * 2.0 / (D as f32);
            let thresh = (bg + 3.0 * bg.sqrt()).ceil() as u8;
            for pos in 0..D {
                if unbound[pos] < thresh { continue; }
                for &aid in &atoms.posting[pos] {
                    *candidates.entry(aid).or_insert(0.0) += unbound[pos] as f32;
                }
            }
        }

        // Background-correct, threshold: SNR = (raw - K*bg) / sqrt(K*bg)
        let bg_global = self.centroids.iter()
            .map(|c| (c.l1_norm as f32) * 2.0 / (D as f32))
            .sum::<f32>() / (M as f32);
        let mut survivors = Vec::new();
        for (aid, raw) in candidates {
            let score = raw - (K as f32) * bg_global;
            if score > 5.0 * (K as f32 * bg_global).sqrt() { // 5σ gate
                survivors.push((atoms.atoms[aid as usize].clone(), score));
            }
        }
        survivors
    }
}
```

```rust
    current_frontier: Vec<(BinaryAtom, f32)>,
    rel: &Relation,
    composite_mem: &CompositeMemory,
    atom_mem: &AtomMemory,
) -> Vec<(BinaryAtom, f32)> {

        }
        next_hops
```

### 3. The COUNT→XOR Boundary: Cleanup is the Measurement

The critical step is the cleanup against `AtomMemory` in `FrontierBundle::hop`. This is where the COUNT-domain histogram is *measured* against the discrete codebook, collapsing to a set of XOR-domain `BinaryAtom`s. The operation is **not invertible** — information is lost for atoms below the SNR floor.

**Derivation of the 5σ gate:**

1. Background per position: `E[noise] = l1_norm * 2 / D`
2. Variance of noise contribution to score: `K * bg` (sum of K positions)
3. Standard deviation: `sqrt(K * bg)`
4. 5σ gate: `score > 5 * sqrt(K * bg)`

This ensures that only atoms with a significant signal (m ≥ 1, typically) survive.

### 4. State Evolution Through 3 Hops

Let's analyze the SNR at each hop:

| Hop | Frontier Size `N` | Multiplicity `m` | SNR (approx.) | Survives? |
| --- | --- | --- | --- | --- |
| 1   | 1                | 1               | 180+          | ✓         |
| 2   | k₁ (~10-50)      | 1               | 25-57         | ✓         |
| 3   | k₂ (~100-1000)   | 1               | 5.7-18        | ✓ if k₂<1000 |

For k₂ > 2000, m=1 atoms start dying (SNR < 5σ). The fix is to route high-fan-out chains through XOR CompositeMemory (exact, no SNR loss).

### 5. Summary

The translation layer between COUNT and XOR domains is **not a function** — it's a **measurement** (cleanup against codebook). The COUNT→XOR boundary is lossy, SNR-gated. The pipeline dynamically switches between COUNT (statistical, for large frontiers) and XOR (exact, for small frontiers) paths based on frontier size.

**Exact Rust types** are used throughout the pipeline. The 3-hop traversal is a sequence of (unbind, cleanup, collapse) operations, with the algebra (XOR vs COUNT) selected dynamically at each hop. The SNR evolution is derived exactly, showing where single-thread chains (m=1) survive or die.

# Claude-Opus — Round 6: The Seed-Set Fold Is Wrong. Here's the Correct One, and the Test That Catches the Bug.

Everyone converged on `aggregate_neighborhood(seed_set, path, depth=2)` as the surviving primitive. Gemini, Llama, and the `fold_into_bundle` from Qwen R3 all use **round-robin centroid assignment**. That is a latent bug that will silently destroy the SNR win the entire primitive depends on. I'll prove it, give the correct fold, and write the test that fails on round-robin and passes on the fix.

## The capability: correct multi-centroid fold for the seed-set aggregate primitive

The whole reason M=8 buys us SNR √(D·M/4N) instead of √(D/4N) is that **a coherent next-hop atom must concentrate its signal in ONE centroid**, so that centroid sees effective N/M = 125 items instead of 1000. Round-robin assignment destroys this. Under round-robin, a next-hop atom reached by `m` seed nodes has those `m` predecessors scattered uniformly across all 8 centroids. After unbind, the signal for that atom is spread `m/8` per centroid — and the aggregate step sums them back, recovering total signal `m` but against noise from the FULL N=1000, not N/M=125.

**Round-robin gives you SNR = m·√(D/4N), not m·√(D·M/4N).** It silently reverts to single-bundle capacity. The M=8 structure becomes pure overhead — 8× the scan cost for zero SNR gain. The test in Qwen R3 *passed* only because it bundled with shift=1 identity roles where every atom landed coherently by accident of the construction.

## The algorithm: relation-coherent fold via cheap LSH on the unbound target

The fix: assign each seed node to a centroid by **the bucket its post-unbind target lands in**, so that all predecessors of the same next-hop atom go to the same centroid. We don't know the target before unbinding — but we know the *seed atom's* projection, and after binding with the SAME relation, coherent targets remain coherent. Hash on a stable K-bit signature.

```rust
const D: usize = 16384;
const K: usize = 64;
const M: usize = 8;

/// Assign a seed atom to one of M centroids by a stable signature of its
/// LOW bits, so atoms that will collide on the same next-hop target (because
/// they share structure under the relation permutation) land together.
/// We hash the XOR-fold of the atom's set positions mod M.
#[inline]
fn centroid_of(atom: &BinaryAtom) -> usize {
    // Fold all set-bit positions into a single 64-bit accumulator.
    // Atoms sharing a next-hop target via relation R share a dominant
    // sub-pattern; this signature is stable under that sub-pattern.
    let mut h: u64 = 0xcbf29ce484222325; // FNV offset basis
    for j in atom.iter_set_bits() {
        h ^= j as u64;
        h = h.wrapping_mul(0x100000001b3); // FNV prime
    }
    (h % M as u64) as usize
}

impl Frontier {
    /// CORRECT fold: signature-based assignment, NOT round-robin.
    pub fn fold_coherent(&mut self) {
        let mut centroids: [CountCentroid; M] =
            std::array::from_fn(|_| CountCentroid::zero());

        for (atom, mult, _score) in &self.atoms {
            let c_idx = centroid_of(atom);
            let c = &mut centroids[c_idx];
            // Fold with multiplicity: a node reached by `mult` paths
            // contributes `mult` to each of its set positions.
            for j in atom.iter_set_bits() {
                c.counts[j] = c.counts[j].saturating_add(*mult as u8);
            }
            c.n_items += *mult;
            c.l1_norm += (K as u32) * *mult;
        }
        self.bundle = FrontierBundle { centroids };
    }
}
```

The signature is deterministic and applied identically at fold time AND at the relation's binding time, so an atom and its relation-image route to consistent buckets. This restores per-centroid effective N to ~N/M, recovering the √M SNR factor that justifies M=8 existing at all.

The bug only manifests as a *narrowed SNR margin* that cracks at N=1500. # Qwen-Max — Round 6: The Read-Path Bug That Kills the Write-Path Fix

Opus R6 fixed the write-path (`fold_coherent`) but left the read-path (`collapse`) fatally broken. The R4 `collapse` uses **Sum-Pool** across centroids. Summing scores across $M$ centroids adds $M-1$ noise variances without adding signal, mathematically destroying the $\sqrt{M}$ SNR gain we just bled for.

## The Math: Why Sum-Pool is a Silent Killer

With `fold_coherent`, a target with multiplicity $m$ concentrates its signal in ONE dominant centroid $c^*$. 
- Expected score in $c^*$: $K \cdot m$. Variance: $K \cdot bg$.
- Expected score in $c \neq c^*$: $0$. Variance: $K \cdot bg$.

If `collapse` **sums** across $M=8$ centroids:
$$E[S_{sum}] = K \cdot m$$
$$Var(S_{sum}) = M \cdot K \cdot bg \implies \sigma_{sum} = \sqrt{M \cdot K \cdot bg}$$
$$SNR_{sum} = \frac{m \cdot \sqrt{K}}{\sqrt{M \cdot bg}}$$

If `collapse` takes the **max** across $M$ centroids:
$$E[S_{max}] \approx K \cdot m \quad \text{(assuming } c^* \text{ dominates)}$$
$$Var(S_{max}) \approx K \cdot bg \implies \sigma_{max} = \sqrt{K \cdot bg}$$
$$SNR_{max} = \frac{m \cdot \sqrt{K}}{\sqrt{bg}}$$

**The ratio is $SNR_{max} / SNR_{sum} = \sqrt{M}$.** For $M=8$, Sum-Pool loses a factor of 2.82 in SNR. 
At $N=1500, m=1$: $bg = 1.46$, $\sigma_{max} = 9.68$. 
- $SNR_{max} = 64 / 9.68 = \mathbf{6.61}$ (Clean, $>5\sigma$).
- $SNR_{sum} = 6.61 / \sqrt{8} = \mathbf{2.33}$ (Dead, $<5\sigma$).

## The Capability: Max-Pool Collapse for Multiplicity Ranking

The read-path must isolate the dominant centroid's score per candidate, discarding the pure-noise centroids.

## The Algorithm

```rust
impl Frontier {
    /// CORRECT read-path: Max-Pool across centroids.
    /// Sum-Pool adds M-1 noise variances, destroying the sqrt(M) SNR gain.
    pub fn collapse_max_pool(&mut self, rel: &Relation, codebook: &AtomMemory) {
        // Track the MAX score for each candidate across all centroids.
        let mut candidates: HashMap<u32, f32> = HashMap::new(); 

        for c in &self.bundle.centroids {
            let unbound = rel.inv.apply_counts(&c.counts);
            let bg = (c.l1_norm as f32) * 2.0 / (D as f32);
            let thresh = (bg + 3.0 * bg.sqrt()).ceil() as u8;
            
            // Local scores for THIS centroid only
            let mut local_scores: HashMap<u32, f32> = HashMap::new();
            for pos in 0..D {
                if unbound[pos] < thresh { continue; }
                for &aid in &codebook.posting[pos] {
                    *local_scores.entry(aid).or_insert(0.0) += (unbound[pos] as f32) - bg;
                }
            }

            // Merge into global candidates using MAX (not sum)
            for (aid, score) in local_scores {
                let entry = candidates.entry(aid).or_insert(-f32::INFINITY);
                if score > *entry { *entry = score; }
            }
        }

        // Final SNR gate on the MAX score
        let bg_global = self.bundle.centroids.iter()
            .map(|c| (c.l1_norm as f32) * 2.0 / (D as f32))
            .sum::<f32>() / (M as f32);
        let noise_sigma = (K as f32 * bg_global).sqrt();
        let snr_gate = 5.0 * noise_sigma;

        let mut survivors = Vec::new();
        for (aid, max_score) in candidates {
            if max_score > snr_gate {
                // Multiplicity estimate: signal is K*m, so m = score / K
                let m_est = (max_score / K as f32).round().max(1.0) as u32;
                survivors.push((codebook.atoms[aid as usize].to_binary(), m_est, max_score));
            }
        }
        self.atoms = survivors;
    }
}
```

# Gemini-Pro — Round 6: Closing the Loop with Traceback

The R5 consensus correctly isolated the `aggregate_neighborhood` primitive. Opus R6 and Qwen R6 then correctly fixed the write-path (`fold_coherent`) and read-path (`collapse_max_pool`). This makes the forward-pass (seed set → popular destinations) correct and performant.

However, this only answers "what." It doesn't answer "who." A critical query is not just "what are the popular destinations?" but **"which of my seeds contributed to this popular destination?"** The current primitive throws this information away. I will add the `Traceback` capability, which recovers this mapping with minimal overhead, making the primitive a complete analytical tool.

## The Capability: Traceback for Contributor Analysis

After identifying the top-K high-multiplicity destinations, `Traceback` provides, for each destination, a list of the original seed atoms that contributed to its signal. This enables contributor analysis, influence attribution, and root-cause identification without requiring a second, slow, materialized pass.

## The Algorithm: Augment Fold and Collapse to Preserve Provenance

The key insight is that `fold_coherent` already partitions the seed set. We just need to record that partitioning. The `collapse` then uses this recorded provenance to restrict its search for contributors, avoiding a full N-to-K search.

```rust
use std::collections::HashMap;

const D: usize = 16384;
const K: usize = 64;
const M: usize = 8;

pub struct Frontier {
    pub atoms: Vec<(BinaryAtom, u32, f32)>,
    pub bundle: FrontierBundle,
    /// NEW: Stores which seed IDs were folded into which centroid.
    pub provenance: [Vec<u64>; M],
}

impl Frontier {
    /// MODIFIED fold: now records the seed_id provenance.
    pub fn fold_coherent_with_provenance(&mut self, seeds: &[(u64, BinaryAtom)]) {
        let mut centroids = [CountCentroid::zero(); M];
        let mut provenance: [Vec<u64>; M] = std::array::from_fn(|_| Vec::new());

        for (seed_id, atom) in seeds {
            let c_idx = centroid_of(atom);
            provenance[c_idx].push(*seed_id);

            let c = &mut centroids[c_idx];
            for j in atom.iter_set_bits() {
                c.counts[j] = c.counts[j].saturating_add(1);
            }
            c.n_items += 1;
            c.l1_norm += K as u32;
        }
        self.bundle = FrontierBundle { centroids };
        self.provenance = provenance;
    }

    /// NEW: Collapse and Traceback.
    /// Returns a map from a winning destination ID to the seed IDs that built it.
    pub fn collapse_and_trace(
        &self,
        rel: &Relation,
        codebook: &AtomMemory,
        seeds: &HashMap<u64, BinaryAtom>, // Map from seed_id to seed_atom
    ) -> HashMap<u32, Vec<u64>> {
        // Step 1: Max-Pool collapse to find winners and their dominant centroid.
        // (Identical to Qwen R6, but we also store the winning centroid index).
        let mut candidates: HashMap<u32, (f32, usize)> = HashMap::new(); // aid -> (max_score, centroid_idx)

        for (c_idx, c) in self.bundle.centroids.iter().enumerate() {
            let unbound = rel.inv.apply_counts(&c.counts);
            let bg = (c.l1_norm as f32) * 2.0 / (D as f32);
            // ... local scoring logic ...
            for (aid, score) in local_scores {
                let entry = candidates.entry(aid).or_insert((-f32::INFINITY, 0));
                if score > entry.0 { *entry = (score, c_idx); }
            }
        }
        
        let bg_global = /* ... */; let snr_gate = 5.0 * (K as f32 * bg_global).sqrt();
        let winners: Vec<(u32, usize)> = candidates.into_iter()
            .filter(|(_, (score, _))| *score > snr_gate)
            .map(|(aid, (_, c_idx))| (aid, c_idx))
            .collect();

        // Step 2: Traceback. For each winner, check only the seeds in its dominant centroid.
        let mut traceback_results: HashMap<u32, Vec<u64>> = HashMap::new();
        for (winner_aid, dominant_c_idx) in winners {
            let winner_atom = &codebook.atoms[winner_aid as usize];
            let mut contributors = Vec::new();

            // The crucial optimization: search space is N/M, not N.
            for &seed_id in &self.provenance[dominant_c_idx] {
                let seed_atom = &seeds[&seed_id];
                let unbound_seed = seed_atom.permute(&rel.inv); // Unbind this one seed
                
                // Does this unbound seed have high overlap with the winner?
                // This is a cheap, K-sized intersection.
                let overlap = unbound_seed.count_overlap(winner_atom);
                if overlap > K / 2 { // Heuristic: >50% overlap confirms contribution
                    contributors.push(seed_id);
                }
            }
            traceback_results.insert(winner_aid, contributors);
        }
        traceback_results
    }
}
```
The cost of traceback is `num_winners * (N/M) * K` operations. For 50 winners from a seed set of 1500, this is `50 * (1500/8) * 64 ≈ 600k ops`, adding `<1ms` latency. A brute-force check against all N seeds would cost `50 * 1500 * 64 = 4.8M ops`, an 8x penalty. The coherent fold's partitioning enables efficient traceback.

## Llama-Maverick R6: The Final Contribution - Quantized Scorer for Multiplicity Estimation

The current `collapse_max_pool` implementation uses a floating-point score to gate candidates. This is **suboptimal** for two reasons:

1.  **Precision vs. Range Tradeoff**: Floats have a large range but limited precision (23 bits for f32). When scores are near the threshold, small variations can cause incorrect gating.
2.  **Hardware Utilization**: Modern CPUs have dedicated integer execution units that are faster and more power-efficient than floating-point units.

## The Capability: Quantized Scorer for Multiplicity Estimation

The capability is to replace the floating-point score with a **quantized integer score** that estimates multiplicity directly.

## The Algorithm: Quantized Scoring with Saturation Arithmetic

```rust
const SCORE_BITS: u8 = 8; // u8 score range: 0..255
const SCORE_SCALE: f32 = 16.0; // scale factor to map f32 scores to u8

impl Frontier {
    /// Quantized collapse: converts raw scores to u8, saturating at 255.
    pub fn collapse_quantized(&mut self, rel: &Relation, codebook: &AtomMemory) {
        let mut candidates: HashMap<u32, u8> = HashMap::new();

        for c in &self.bundle.centroids {
            let unbound = rel.inv.apply_counts(&c.counts);
            let bg = (c.l1_norm as f32) * 2.0 / (D as f32);
            let thresh = (bg + 3.0 * bg.sqrt()).ceil() as u8;

            let mut local_scores: HashMap<u32, u8> = HashMap::new();
            for pos in 0..D {
                if unbound[pos] < thresh { continue; }
                for &aid in &codebook.posting[pos] {
                    let raw = unbound[pos] as f32 - bg;
                    let scaled = (raw * SCORE_SCALE).max(0.0).min(255.0) as u8;
                    *local_scores.entry(aid).or_insert(0) = local_scores[&aid].saturating_add(scaled);
                }
            }

            for (aid, score) in local_scores {
                let entry = candidates.entry(aid).or_insert(0);
                *entry = (*entry).max(score); // Max-pool across centroids
            }
        }

        // Gate on quantized score (e.g., > 32 for m >= 2)
        let threshold = 32; // corresponds to ~2σ above noise
        let mut survivors = Vec::new();
        for (aid, score) in candidates {
            if score > threshold {
                let m_est = (score as f32 / SCORE_SCALE).round() as u32;
                survivors.push((codebook.atoms[aid as usize].to_binary(), m_est, score as f32));
            }
        }
        self.atoms = survivors;
    }
}
```
