## Round 4: Reconsidering Chained Residuals with Semantic Embeddings

The self-annihilation problem is a significant challenge for algebraic composition in Hyperdimensional Computing (HDC). However, with the introduction of semantic embeddings from Debate D, the vectors representing the same concept in different contexts are distinct. ### 1. Understanding the Chained Residual Approach

The chained residual approach involves computing the XOR of two vectors representing the same relation in different contexts. For the "father" relation:
- `father_c1`: The vector representation of "father" in context 1.
- `father_c2`: The vector representation of "father" in context 2.

The residual is calculated as:
\[ R = \text{father}_c1 \oplus \text{father}_c2 \]

### 2. Deriving the Expected Jaccard Similarity

To assess whether `R` can represent "grandfather", we need to calculate the Jaccard similarity between `R` and `grandfather`.

#### Assumptions:
- Dimension `D = 16384`.
- Sparsity `k = 64` for all vectors.
- The vectors `father_c1` and `father_c2` are `k`-sparse and have a certain Jaccard similarity due to their semantic similarity.
- `grandfather` is an independent `k`-sparse vector.

#### Step 1: Calculate the Size of `R`

Given that `father_c1` and `father_c2` are `k`-sparse:
\[ |R| = |(\text{father}_c1 \cup \text{father}_c2) - (\text{father}_c1 \cap \text{father}_c2)| \]
\[ |R| = k + k - 2|\text{father}_c1 \cap \text{father}_c2| \]
\[ |R| = 2k(1 - J(\text{father}_c1, \text{father}_c2)) \]

\[ E[|R \cap \text{grandfather}|] = \frac{|R| \times k}{D} \]

\[ E[J(R, \text{grandfather})] = \frac{E[|R \cap \text{grandfather}|]}{|R| + k - E[|R \cap \text{grandfather}|]} \]

- `D = 16384`

    expected_intersection = (size_R * k) / D
    union_size = size_R + k - expected_intersection

**Key insight, new to this debate:** A genuine composition rule `src ∘ via = derived` predicts triples the discovery set *did not contain*. ---

## The Algorithm: Split-Half Cross-Prediction

Split the supporting chains into two disjoint halves. Crystallize the rule from half A. Test whether it predicts the *derived* endpoints in half B — chains it never saw. Genuine rules transfer; poisoned clusters don't.

```rust
/// Promotion gate. Returns a confidence in [0,1]. Human reviews only c > 0.6.
/// Adversarial support fails because crafted triples don't generalize off-set.
pub fn promotion_confidence(
    chains: &[(Sparse /*A*/, Sparse /*C, observed derived endpoint*/)],
    codec: &EventCodec,
    dict: &[Sparse],
) -> f64 {
    let n = chains.len();
    if n < 14 { return 0.0; } // need >=7 per half (N_safe)

    let mid = n / 2;
    let (train, test) = chains.split_at(mid);

    // Crystallize rule atom from TRAIN residuals only (R3 majority-threshold).
    let residuals: Vec<Sparse> = train.iter()
        .map(|(a, c)| a.xor(c))           // chain residual A⊕C
        .collect();
    let rule = match discover_rule(&residuals, 0.5) {
        Some(r) => r,
        None => return 0.0,
    };

    // Apply rule to TEST sources, check predicted endpoint matches observed.
    let mut hits = 0usize;
    for (a, c_observed) in test {
        let predicted_raw = a.xor(&rule);          // Ĉ = A ⊕ R
        let recovered = hopfield_cleanup(&predicted_raw, dict);
        // Genuine rule: recovered == observed endpoint. Poison: random.
        if recovered.jaccard(c_observed) > 0.5 {
            hits += 1;
        }
    }
    hits as f64 / test.len() as f64
}
```

**Why this kills the poisoning vector specifically:** the 55-composite / 15-triple attack from Debate E manufactures *internally consistent* support. But the attacker controls only the chains they injected. Split-half forces the rule learned from one injected subset to predict endpoints in a *different* subset. If the attacker poisoned both halves identically, that's no longer 15 triples — it's 28+, doubling attack cost per rule, and the split can be randomized across `k` folds to multiply it further. Genuine emergent rules pay zero penalty: real `metabolized_by ∘ inhibits` transfers to held-out drugs by construction.

**Cost:** one extra `discover_rule` (O(N·k) ≈ 1.8K ops) + `test.len()` cleanups. Negligible — it runs in the *batch* job, never the query path. This is the cheap mechanical filter that makes "human-in-the-loop" tractable: it converts thousands of raw suggestions into a ranked shortlist where confidence correlates with genuineness, not with support volume.

/// Promotion gate. Returns a confidence in [0,1]. Human reviews only c > 0.6.
/// Adversarial support fails because crafted triples don't generalize off-set.
pub fn promotion_confidence(
    chains: &[(Sparse /*A*/, Sparse /*C, observed derived endpoint*/)],
    codec: &EventCodec,
    dict: &[Sparse],
) -> f64 {
    let n = chains.len();
    if n < 14 { return 0.0; } // need >=7 per half (N_safe)

    let mid = n / 2;
    let (train, test) = chains.split_at(mid);

    // Crystallize rule atom from TRAIN residuals only (R3 majority-threshold).
    let residuals: Vec<Sparse> = train.iter()
        .map(|(a, c)| a.xor(c))           // chain residual A⊕C
        .collect();
    let rule = match discover_rule(&residuals, 0.5) {
        Some(r) => r,
        None => return 0.0,
    };

    // Apply rule to TEST sources, check predicted endpoint matches observed.
    let mut hits = 0usize;
    for (a, c_observed) in test {

        let recovered = hopfield_cleanup(&predicted_raw, dict);

        if recovered.jaccard(c_observed) > 0.5 {
            hits += 1;
        }
    }
    hits as f64 / test.len() as f64
}

pub struct FractalCascade {

    subclusters: HashMap<u64, Sparse>, // 100K centroids (k=256)

    // so we hash the first 48 indices of the sorted sparse vector).

    /// Extracts a stable routing key from the Anchor portion of a sparse vector.
    fn anchor_hash(&self, atom: &Sparse) -> u64 {
        // Take the first 48 indices (the Anchor tier from Debate D)
        let anchor_slice = &atom.0[..48.min(atom.0.len())];
        let mut hasher = std::collections::hash_map::DefaultHasher::new();
        anchor_slice.hash(&mut hasher);
        hasher.finish()
    }

    /// The rescue mechanism. Called when primary k=64 cleanup fails.
    pub fn cascade_rescue(&self, noisy_query: &Sparse, dict: &[Sparse]) -> Option<Sparse> {
        // 1. Primary attempt fails (Jaccard < 0.30 cliff)
        let primary_match = hopfield_cleanup(noisy_query, dict);

            return Some(primary_match); // No rescue needed

        let query_hash = self.anchor_hash(noisy_query);

        // 4. Re-run Hopfield Cleanup with the boosted prior

        let rescued_match = hopfield_cleanup(&boosted_query, dict);

        } else {
            None // Fell off the k=256 cliff, route to k=1024 category (omitted for brevity)
        }
    }
}
